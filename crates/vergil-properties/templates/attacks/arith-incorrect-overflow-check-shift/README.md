# arith-incorrect-overflow-check-shift

The canonical "why bitvector theory matters" Vergil demo. The real-world instance is **Cetus Protocol** (May 22, 2025, ~$223M, Sui) — the contract's overflow guard compared the input against `0xFFFFFFFFFFFFFFFF << 192` (a constant equal to `2^256 - 2^192`, a much looser threshold than intended). Inputs in `[2^192, 2^256 - 2^192)` passed the guard and overflowed the subsequent `value << 64`, silently truncating the high 64 bits and letting the attacker mint enormous liquidity for ~1 token. Vergil's negation property requires that `(value << 64) >> 64 == value` — i.e., the shift is lossless. The vulnerable fixture's wrong threshold yields a counterexample under Z3's bitvector theory (LIA cannot reason about shift overflow exactly); the clean fixture's `value < 1 << 192` threshold verifies. See `manifest.yaml` for references (OWASP SC09:2026, Halborn Cetus post-mortem) and `notes/attack-patterns.md` §3.4.
