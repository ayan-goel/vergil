# arith-overflow-underflow-unchecked

Detects integer wraparound inside `unchecked { ... }` blocks (and pre-0.8 contracts). The canonical real-world instance is the **BeautyChain (BEC) `batchOverflow`** (Apr 2018, Ethereum) — a multiplication overflow minted vast token amounts and crashed the price. Vergil's negation property requires that arithmetic results remain representable in their declared type; the Halmos encoding asserts the result of an addition is no smaller than either input, which is impossible under wraparound. The vulnerable fixture wraps `a + b` in `unchecked { ... }`; the clean fixture relies on Solidity 0.8 default-checked arithmetic, which reverts on overflow and renders the assertion vacuously true. See `manifest.yaml` for references (SWC-101, OWASP SC09:2026) and the catalog entry `notes/attack-patterns.md` §3.1.
