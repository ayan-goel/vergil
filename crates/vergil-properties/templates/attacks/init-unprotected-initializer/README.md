# init-unprotected-initializer

Detects the most common upgradeable-contract pitfall: a public `initialize()` callable by anyone, with no one-shot guard. The canonical instance is **Audius** (Jul 2022, ~$6M, Ethereum) — the proxy's initializer was reachable post-deployment, an attacker re-initialized governance, and a treasury-draining proposal followed. Vergil's negation property requires that the initializer run at most once and that the first caller's claim to ownership/admin survive. The Halmos encoding has the check contract initialize the target legitimately (becoming the first owner); a separate Attacker contract then attempts a second initialize. In the vulnerable fixture the attacker seizes ownership; in the clean fixture (with a `bool private initialized` guard) the second call reverts. See `manifest.yaml` for references (SWC-105, OWASP SC10:2026) and the catalog entry `notes/attack-patterns.md` §2.1.
