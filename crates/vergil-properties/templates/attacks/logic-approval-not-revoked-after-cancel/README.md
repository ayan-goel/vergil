# logic-approval-not-revoked-after-cancel

Detects vesting / airdrop / token contracts whose `cancel` path fails to revoke a previously-granted allowance. The canonical instance is **Hedgey Finance** (Apr 19, 2024 — $42.6M on Arbitrum + $2.1M on Ethereum) — the cancel function left per-plan allowances live, and the attacker drained via `transferFrom` against the stale approvals. Vergil's negation property requires that cancel zero any allowances it earlier granted. The Halmos encoding creates a plan with a non-zero allowance, calls cancelPlan, and asserts the allowance is zero. Pairs with `input-missing-parameter-validation` — the Hedgey fixture surface is shared because the production bug had both flaws. See `manifest.yaml` for references (OWASP SC02:2026, Halborn post-mortem) and `notes/attack-patterns.md` §7.1.
