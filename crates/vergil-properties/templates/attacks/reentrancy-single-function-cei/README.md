# reentrancy-single-function-cei

The Checks-Effects-Interactions canon. The canonical real-world instance is **The DAO** (Jun 17, 2016, ~3.6M ETH / ~$60M at the time, Ethereum) — `splitDAO()` made an external value transfer before updating the depositor's balance, so a malicious recipient's fallback recursively withdrew until the balance state was finally written. Vergil's negation property requires that no state-change occurs after an external call inside a single function's execution; the Halmos encoding casts the check contract as the attacker, sets a `reentering` flag, invokes the target's action, and uses its own `receive()` callback to attempt a single re-entry. The vulnerable fixture has no guard so the re-entry succeeds and the tracked counter double-increments; the clean fixture's hand-rolled `nonReentrant` modifier reverts the re-entry. See `manifest.yaml` for full references (SWC-107, OWASP SC08:2026) and the catalog entry `notes/attack-patterns.md` §4.1.
