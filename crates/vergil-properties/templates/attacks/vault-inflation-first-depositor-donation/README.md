# vault-inflation-first-depositor-donation

Detects the ERC-4626 first-depositor / donation inflation class. The attacker deposits 1 wei to mint 1 share, donates a large asset balance directly to the vault to inflate the share price, then waits for a victim to deposit — whose small contribution rounds to zero shares while their assets are absorbed. The class was widely documented in 2022–2023 and prompted OpenZeppelin's virtual-shares offset mitigation (`_decimalsOffset` since v4.9). Vergil's negation property requires that any non-zero asset deposit yield non-zero shares; the Halmos encoding follows the attack sequence (deposit 1 → donate inflation → victim deposit) and asserts `sharesOut > 0`. The vulnerable fixture absorbs the assets with zero shares; the clean fixture's `require(sharesOut > 0)` reverts. The encoding bounds the donation < 2^128 so the solver stays tractable; the full OZ virtual-shares mitigation is a richer V2 enhancement. See `manifest.yaml` for references (OWASP SC07:2026) and `notes/attack-patterns.md` §5.1.
